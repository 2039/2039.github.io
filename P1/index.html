2: Administrative details

  • Client name: Laura Orem
  • Contact person: Ina P. Sum

Our (imaginary) client is a researcher in Lorem Ipsum at the Dolor Sit Amet Institute in Trondheim. Laura has spent considerable time investigating and learning about this language, but has come to a halt. She has requested that we make a website that demonstrates the peculiar aspects of this language, with the hope that this may raise awareness and aid her research.

3: Purpose and goals

The purpose will be to make a website that is accessible by as many as possible, and to reach as many as possible. With our unique approach we hope to reach at least 5 visitors (not necessarily different ones) in the next decade. Anyone with even the slightest knowledge of language should immediately be captivated by our website.

4: Audience

Lorem ipsum enthuisiasts everywhere. We hope to popularize and bring new audience to experience this amazing language and to realize its importance in bringing the world to where it is today. We believe that for every new person who learns about this language will live a better life and get smoother skin.

5: The content of the site and how it is organized

Front page

This should be the landing page and should contain a sitemap leading users deeper into the Lorem Ipsum universe.

Quiz page

This should contain questions which lets the user select and submit and receive a score based on values.

Translator page

This should translate user input into Pig Latin.

Library

This should contain a page listing different (generated) papers, and lets the user select and read an article. This should also include a download-button so that the user can download and/or print an article.

About

This should contain information about the project.

6: Functional and non-functional requirements

  • The site should have a Pig Latin translator.
  • The site should answer questions about Lorem Ipsum.
  • The user should be able to obtain (download) information about Lorem Ipsum.
  • The user should be able to reach any page in less than 4 clicks.
  • The site should be accessible for all groups of people on all devices (i.e. mobile and computer).
  • The site should be lightweight.
